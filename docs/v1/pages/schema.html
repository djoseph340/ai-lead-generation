Data Schema

Lead Output Format

Every processed lead outputs a consistent JSON structure, regardless of input format.

Example Lead Object

{ "name": "Donald Smith", "company": "Baylor Inc", "email": "donald@baylor.com", "industry": "Healthcare", "employees": 200, "message": "Looking for supply chain automation", "lead_score": 75, "priority": "Medium", "tier": "Tier B", "reason_codes": "weak_role_fit", "timestamp": "2026-07-04T14:23:45Z" }

Field Definitions

Field Type Description
name string Lead contact name from form
company string Company name
email string Lead email address
industry string Industry classification (e.g., Healthcare, Finance)
employees integer Company employee count
message string Lead's message or inquiry text
lead_score integer Calculated score (0-95)
priority string High / Medium / Low
tier string Tier A / B / C (unqualified only)
reason_codes string Why lead is unqualified (unqualified only)
timestamp ISO8601 When lead was received (UTC)

Google Sheets Storage

Sheet 1: Leads (Qualified)

High-priority leads (score ≥ 80) ready for sales outreach.

Columns: name, company, email, industry, employees, message, lead_score, priority, timestamp
Sheet 2: Unqualified (Below Threshold)

Lower-priority leads (score < 80) with classification info.

Columns: name, company, email, industry, employees, message, lead_score, priority, tier, reason_codes, timestamp

Reason Codes

These explain why an unqualified lead didn't make the cut:

insufficient_score

Lead's signals didn't add up to 80 points.

weak_role_fit

Role or title doesn't match our target personas.

wrong_industry

Industry not on our target list.

company_size_mismatch

Company too small or too large for fit.

Storage Details

Unqualified Leads Real-time append
