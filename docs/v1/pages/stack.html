n8n

Workflow orchestration platform. Handles webhooks, node logic, and integrations.

Role: Core engine
Webhook (HTTP POST)

Entry point for form submissions from Tally, Typeform, or similar platforms.

Role: Lead intake
JavaScript

Custom scoring engine and field extraction logic in n8n Code nodes.

Role: Business logic
Google Sheets API

Real-time data persistence. Appends lead records to Leads and Unqualified tabs.

Role: Data storage

Why These Choices?

n8n (vs. Zapier, Make)

Open-source, self-hosted option. Full control over workflows and logic. Good for complex conditional routing.

Google Sheets (vs. Database)

Fast to set up, non-technical users can access and filter. Good for v1. Can migrate to SQL later if needed.

JavaScript (vs. Python)

n8n's native language for Code nodes. No external dependencies or deployments needed.

Rule-Based (vs. ML)

Fast, interpretable, and doesn't require training data or retraining. Easy to adjust weights on the fly.

Deployment & Hosting

  • n8n Instance: Can be self-hosted or use n8n Cloud
  • Google Sheets: Cloud-hosted. Requires API credentials.
  • Webhook URL: n8n provides the endpoint for form platforms to POST to
  • No servers to manage: Forms → n8n → Sheets (fully managed)

Dependencies & Configuration

  • n8n account (with webhook enabled)
  • Google Cloud project (for Sheets API access)
  • Google Sheets authentication (OAuth 2.0)
  • Form platform webhook (Tally, Typeform, etc.)
